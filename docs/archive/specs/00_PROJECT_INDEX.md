# CredenceAI Project Index

## Project Summary

CredenceAI is a free-first, open-source, AI-assisted search intelligence platform. Its job is to transform messy public information into verified, deduplicated, entity-aware, source-traceable intelligence.

## Core Operating Model

```text
Discover broadly
→ normalize strictly
→ score aggressively
→ deduplicate early
→ enrich intelligently
→ crawl safely
→ extract accurately
→ validate critically
→ index only trusted data
→ benchmark continuously
```

## Refined Product Direction

CredenceAI should not be positioned as a generic search engine. The stronger position is:

> CredenceAI is a trusted intelligence layer that converts noisy web and open-source data into verified, evidence-backed knowledge for analysts, enterprises, researchers, and AI systems.

## Product Maturity Model

| Iteration | Product capability |
|---:|---|
| 1 | CredenceAI can accept jobs, search free sources, normalize, store, index, and return results. |
| 2 | CredenceAI can score quality, deduplicate results, resolve entities, and attach trust decisions. |
| 3 | CredenceAI can safely crawl, extract content, score freshness, and index trusted documents. |
| 4 | CredenceAI can use governed AI agents to validate ambiguous cases and generate intelligence cards. |
| 5 | CredenceAI can scale, benchmark, optimize latency/cost, and run in production. |

## Mandatory Design Rules

1. Do not crawl before scoring.
2. Do not index before validation.
3. Do not use paid APIs until benchmark evidence proves value.
4. Do not use Playwright unless the page value justifies browser cost.
5. Do not let AI agents bypass deterministic safety and compliance gates.
6. Do not merge entities without confidence scoring.
7. Do not ship source adapters without contract tests.
8. Do not call any architecture change an improvement without benchmark evidence.

## Final Expected State

By the end of Iteration 0.5, CredenceAI should support:

- Free-first source discovery.
- Intent-based source routing.
- Source normalization.
- Quality and trust scoring.
- URL, content, media, and entity deduplication.
- Entity resolution.
- Safe selective crawling.
- Document extraction and freshness scoring.
- Evidence-backed summaries.
- Intelligence cards.
- RAG-ready datasets.
- Benchmarking and A/B testing.
- Fast, Standard, and Deep execution modes.
- Production observability and alerting.
